Key Insights

Haiku dominates volume (39%) but costs only $186 total — used for fast, lightweight tasks where low latency matters most.
Opus models drive 73% of total cost ($4,258 combined) despite representing only 42% of requests — used for complex, high-value tasks.
Avg API latency: 8.98 s — Opus models average ~10-11 s versus Haiku's ~5.5 s, reflecting the quality/speed trade-off.
Sonnet-4-6 is newest and underused (2%) — adoption expected to grow as teams migrate from older Sonnet/Opus versions.

Forecasting Methodology

A linear regression model (numpy polyfit, degree 1) was applied to daily event counts to project the next 7 days of usage. The model is fitted on the full date range selected in the dashboard and clamps all predictions to ≥ 0.

Strategic Recommendations

1. Control Opus costs — Opus models consume 73% of the budget. Route simple/repetitive tasks to Haiku with an automatic model-selection policy to reduce cost by an estimated 30–40%.
2. Fix rate limiting (429 errors) — 299 avoidable failures. Implement exponential back-off and a request queue to eliminate this error class entirely.
3. Grow Platform Engineering adoption — 13 users vs. 25 in Frontend. Run targeted workshops showing Claude Code for Terraform, CI/CD, and infra automation.
4. Migrate to newer models — Sonnet-4-6 and Opus-4-6 are available but underused. Older versions (4-5) still generate 43K+ requests/month in costs.
